# Guide: Evaluating the AI Job Agent System & Model

To evaluate the effectiveness of this system, you need to measure two layers: the **LLM Model & Prompt Accuracy** and the **End-to-End Pipeline Performance**. Below is a systematic evaluation blueprint.

---

## 1. LLM Model & Prompt Evaluation

This evaluates the underlying Llama models (`llama-3.1-8b-instant` or similar) and prompts configured inside `src/lib/agents/`.

### A. CV Parsing Accuracy (Precision & Recall)
* **What to Test**: Upload a set of 5-10 distinct resume styles (academic, timeline-based, skill-based, single/multi-column PDFs).
* **Metrics**:
  * **Extraction Match**: Compare parsed skills, experiences, and names against the original CV document.
  * **JSON Integrity**: Check if the LLM output is parsed successfully without returning syntax errors.

### B. Job Matching Alignment
* **What to Test**: Navigate to the jobs list and review the **Match Score %** generated by the model.
* **Metrics**:
  * **True Positives**: Does the LLM score jobs high (e.g., >80%) when they match your CV skills?
  * **True Negatives**: Does the LLM correctly filter out non-matching fields or junior positions if you set senior preferences?

### C. Interview Q&A Quality
* **What to Test**: Run through the mock interview. Type a bad answer, a half-correct answer, and a stellar answer.
* **Metrics**:
  * **Score Sensitivity**: Does a bad answer score low (1-3/10) and a stellar answer score high (8-10/10)?
  * **Actionability**: Are the recommendations in the final report constructive and specific to the questions asked, or are they generic?

---

## 2. Agent Pipeline & Integration Evaluation

This evaluates the automation background loops (active scraper, mail parser, and notifications).

### A. Web Scraper Matching Rate
* **What to Test**: Let the active pipeline run or review the summary emails.
* **Metrics**:
  * **Scraping Volume**: Number of job posts harvested per cycle from target boards.
  * **Match Ratio**: Percentage of scraped jobs that qualify for automatic application.

### B. Email Classification Sensitivity
* **What to Test**: Use the **Simulate Company Reply** box on the dashboard to test these types of messages:
  1. A clear interview invite (should change status to `Interview`).
  2. An acknowledgement stating "We received your application and will review it" (should change status to `Under Review`).
  3. A standard rejection notice (should change status to `Rejected`).
* **Metrics**:
  * **Classification Accuracy**: Percentage of simulated or real emails mapped to the correct pipeline status.

### C. Inbox Checking Reliability (IMAP)
* **What to Test**: Verify that the checker skips emails dated before the application date and marks incoming matches as read (`\Seen` flag).

## 📊 Summary Evaluation Scorecard Template

Use this rubric to score your model during demonstrations:

| Feature Dimension | Test Case | Success Criteria | Target Metric |
| :--- | :--- | :--- | :--- |
| **CV Extraction** | Standard PDF Upload | Structured JSON maps exactly to career fields | > 95% Parsing Rate |
| **Job Match Scoring** | High/Low Relevance Jobs | High matches get 80-100%, irrelevant get < 50% | > 90% Classification Accuracy |
| **Email Reply Parser** | Under Review vs Interview | Correctly separates status changes | 100% Classification |
| **Mock Interview** | 5-Question Session | Locked interface + final feedback card | 100% Session Completion |
| **Email Summary** | Background Cron Cycle | Standardized text format with stats and last 5 | 100% Delivery |
